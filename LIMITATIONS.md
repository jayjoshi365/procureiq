# ProcureIQ — Limitations

This document is an honest description of what ProcureIQ does not do, does not guarantee,
and is not suitable for. Read this before using the tool for real sourcing decisions.

---

## Architecture Limitations

### SQLite / Single-User
- The default database is SQLite. It supports one concurrent writer.
- The app is designed for single-user personal use and portfolio demonstration.
- It is **not suitable** for concurrent multi-user deployment without migrating to PostgreSQL
  and running behind a proper WSGI/ASGI server with connection pooling.
- WAL mode is enabled for improved read concurrency, but this does not make it production-safe
  for multi-user write workloads.

### No ERP Integration
- There is no live connection to SAP Ariba, Coupa, Ivalua, Oracle Procurement Cloud, or any
  other enterprise procurement platform.
- Spend data, supplier data, and contract data are entered manually, via CSV import, or via Excel upload.
- CSV import supports supplier name, ticker, price, financial fields, and all 9 evaluation dimension scores (up to 20 rows). Any dimension scores left blank in the CSV must be set manually after import.
- The FastAPI layer provides endpoints for data ingestion, but there is no certified connector
  to any enterprise system.

### Session Persistence
- Session data is stored in SQLite via a key-value config table.
- If the Streamlit server restarts, in-memory session state is lost unless explicitly saved.
- The Streamlit UI requires login via a custom `auth.py` bcrypt login gate. This is a demo-grade credential check — not enterprise identity management. It does not provide MFA, SSO, or role-based access control. Sessions are in-memory only and do not persist across browser refreshes or server restarts.

## Data Limitations

### Fallback / Illustrative Data
- When no Anthropic API key is configured, supplier discovery returns **illustrative data** sourced
  from a static knowledge base (curated market leaders per category).
- These results are clearly labeled with "⚠️ ILLUSTRATIVE" banners throughout the UI.
- Illustrative scores, rankings, and supplier descriptions are **estimates only** and have not been
  validated against live data sources.
- Do not use illustrative discovery results to support a real sourcing decision.

### Financial Health Scores
- For **public companies** with a US stock ticker, the Financial Health score (0–100) is derived
  from SEC EDGAR/XBRL annual filing data: revenue growth, profit margin, and debt-to-assets ratio.
  The score is labeled "SEC EDGAR/XBRL" (all three metrics) or "Partial EDGAR + User" (1–2 metrics).
  Users can override the EDGAR score with their own manual assessment.
- For **private companies** (no ticker, or ticker not found in SEC EDGAR), the score is derived from
  qualitative user-entered answers (e.g., "Years in Business", "Revenue Trajectory").
- EDGAR data reflects the most recent 10-K annual filing. It is not real-time and may lag by up to
  12 months. Not all public companies have complete XBRL data in SEC EDGAR.
- The score card displays a freshness badge showing the SEC filing period date and a color-coded
  staleness indicator (green ≤12 months, amber 13–18 months, red >18 months). Treat amber and red
  badges as a prompt to verify recency before final award — not as a disqualifier.
- In all cases, do not treat the Financial Health score as a substitute for credit checks,
  Dun & Bradstreet reports, auditor assessments, or professional financial due diligence.

### Market Intelligence
- Live market data (stock prices, company overviews) requires Alpha Vantage and/or SEC EDGAR API keys.
- Without API keys, market intelligence is static and may be out of date.
- USASpending.gov data reflects US federal government contracts only. It is not a proxy for
  commercial market position and may be irrelevant for private-sector sourcing decisions.

## AI / LLM Limitations

### AI Provider Support
- All AI agents (supplier discovery, CFO narrative, risk analysis, intake coaching) require Claude (Anthropic).
- The settings panel exposes alternative providers (OpenAI, DeepSeek, Grok) for experimental text generation use only. These providers cannot invoke tool-use agents. Selecting them disables supplier discovery and CFO Challenge AI generation.
- Alternative providers are labeled "LIMITED — AGENTS DISABLED" in the UI and are collapsed behind an experimental expander.

### LLM Output Requires Human Review
- All outputs from the LLM (supplier discovery rationale, intake analysis, contract language suggestions)
  are generated by a large language model and **have not been validated** against external sources.
- LLM outputs may contain hallucinated supplier names, incorrect market facts, or outdated information.
- A qualified procurement professional must review and validate all AI-generated outputs before
  using them in a sourcing decision.

### No Real-Time Knowledge
- The LLM's training data has a knowledge cutoff. Recent M&A activity, new entrants, or market shifts
  may not be reflected in discovery outputs.
- Always supplement AI discovery with current analyst reports and supplier due diligence.

## Compliance Limitations

### Sanctions Screening
- The sanctions check feature is a **stub and demonstration only**.
- It does not perform actual OFAC SDN list screening, SAM.gov debarment checks, or any certified
  compliance screening.
- Do not use ProcureIQ's sanctions feature as a substitute for certified compliance tooling.
- Before awarding a contract, perform sanctions screening using a certified data source
  (e.g., Dow Jones Risk & Compliance, LexisNexis WorldCompliance, or an OFAC-licensed screening service).

### Not Legal Advice
- Recommendation outputs, risk flags, negotiation playbooks, and contract language suggestions are
  informational only and do not constitute legal advice.
- Consult qualified legal counsel before executing contracts or making procurement commitments
  based on ProcureIQ outputs.

### Not an Audit Tool
- The audit log feature captures user actions for informational purposes.
- It is not a certified audit trail and does not meet SOX, GDPR, ISO 27001, or other compliance
  framework audit requirements.

## Security Limitations

### Demo-Grade Security
- ProcureIQ has demo-grade security appropriate for personal use and portfolio demonstration.
- It is **not suitable** for processing sensitive supplier data, PII, or confidential commercial
  information without additional hardening.
- Specific limitations:
  - SQLite stores data in a local file with no encryption at rest by default.
  - The Streamlit login gate (`auth.py`) is a demo-grade bcrypt credential check — not enterprise identity management. It provides no MFA, SSO, role-based access control, or persistent sessions.
  - JWT authentication is available for the FastAPI layer but is optional and requires manual setup.
  - The login endpoint has in-memory rate limiting (5 failures / 5 min per IP), but it resets on server restart and does not protect distributed deployments.
  - The `SECRET_KEY` for JWT signing must be set manually — it is not auto-generated.

### No Multi-Tenancy
- All data in a single instance is shared. There is no tenant isolation, user-level access control,
  or data separation between different users of the same instance.

---

## What ProcureIQ Is Suitable For

- Personal portfolio projects demonstrating procurement technology concepts.
- Structured thinking support for individual procurement professionals.
- Interview and demo walkthroughs of a procurement decision-support workflow.
- Learning and experimentation with AI-assisted sourcing concepts.
- Prototyping a procurement tool before building a production version.

## What ProcureIQ Is Not Suitable For

- Processing real supplier PII or commercially sensitive contract data.
- Making binding sourcing decisions without independent validation.
- Multi-user enterprise deployment without significant additional engineering.
- Replacing certified compliance, legal, or financial advisory services.
- Any context where the limitations above would create business, legal, or compliance risk.
